통신을 하려면
ip, mac address, port 가 필요하다
mac address : 랜카드주소, 이더넷 주소를 말한다. 
              각자의 랜카드에는 16진수 쌍으로 이루어진 전세계에서 유일한 값이 저장되어 있다. 
ip : 컴퓨터를 식별하기 위해 논리적으로 부여하는 코드
      4byte, 1byte.1byte.1byte.1byte 형태인데
      외우기 어려워서 대신에 도메인 이름을 사용
      wwww.daum.net(도메인이름) ---------> DNS 서버( 도메인 네임 서버) 
              <--------------------------- IP를 알아온다

IP는 통신을 하고자 하는 상대방의 단말장치 ( 컴퓨터 등 )
상대방의 단말장치에 가서 만나야 할 프로세스가 누구인지 ... 포트번호 

포트번호 : 프로세스 식별값이라고 흔히 말하는데
           엄밀히 말하면 소켓의 식별값이다. 
           oracle, http 등등 누구와 만날 건지를 정하는 값
           ex ) HTTP 는 80번 포트를 사용 : http://www.daum.net:80 원래는 이렇게 포트번호를 써야함
                그런데 HTTP의 포트번호가 80이라는 건 모두가 알기 때문에 암묵적으로 생략

소    켓 : 통신을 할 때 LAN카드 내부 구조
           개발자로 하여금 소켓 이하의 내용은 알지 않아도 개발을 가능하도록 함
           ex) 과거 console.log를 개발한 사람은 
               그래픽카드의 구조부터 모니터 화면에 데이터를 띄우는 
               방법까지 모두 파악하고 있어야 가능했음
          버클리의 소켓 라이브러리가 최초의 네트워크라이브러리 
          소켓 라이브러리는 소켓 내의 변수와 함수로 구성된 라이브러리
          소켓이 구성된 이후는 실행은 알아서

서    버 : 는 특정한 장비에 붙여진 이름이 아닌
           서버 기능을 제공하는 장치에 그때 그때 붙게되는 이름.
           그렇지만 현실적으로 서버컴퓨터로서 작동하기 위해서는 높은 사양을 요구하기 때문에
           서버컴퓨터는 실제로는 제한적으로 사용됨.
